What we are building

RiskTrix is focused on practical risk heatmaps, not generic reporting filler.

The current site presents RiskTrix as a dedicated risk heatmap tool for teams that need to show likelihood and impact clearly in meetings, reports, and stakeholder updates.

Our work right now is centred on making the matrix easier to read, easier to explain, and easier to use as a decision-making artefact. That means stripping away placeholder marketing language and only showing the product direction that is already real.

As the product scope is confirmed with the data and backend teams, this site can expand to document the released feature set. For now, the public-facing message stays intentionally simple and aligned with the current sprint goal.

Current focus

Prepared for the next sprint review

This version of the website is being prepared for public release. The priority is to present RiskTrix accurately as a risk heatmap product and remove any content that suggests features, integrations, or workflows that are not yet confirmed.

The messaging now reflects the product's current state: a focused tool for communicating risk position visually, with clean presentation and straightforward contact pathways for follow-up conversations.

Release principle

Only show what already exists

Before launch, every page should match the latest client-approved social content and only describe the functionality that is currently available. This keeps the public website credible, consistent, and ready for stakeholder review.
